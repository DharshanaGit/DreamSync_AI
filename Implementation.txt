Implementation Plan - DreamSync Smart Planner
DreamSync is a smart planner for students and working professionals to balance long-term goals (e.g., placements, competitive exams, coding) with college or work. It features goal management, task tracking, habit tracking, a smart daily schedule auto-generator, and progress analytics.

User Review Required
IMPORTANT

Database fallback: We will configure the Django settings to support PostgreSQL by default (useful for production/user's preference) but fallback automatically to SQLite if environment variables or database connection fails. This ensures the application works immediately out-of-the-box.
Authentication: We will use JWT (JSON Web Tokens) with standard cookies or localStorage. To make the Django-rendered frontend feel integrated, we will serve pages as standard Django views and use client-side JavaScript to inspect JWTs, handle authorization, and dynamically make API requests to Django REST Framework (DRF) endpoints.

Proposed Changes
We will create a Django project structure named dreamsync_project and a main app named planner.

1. Backend: Django Project & Apps
[NEW] 
settings.py
Setup Django configuration.
Integrate rest_framework and rest_framework_simplejwt.
Configure SQLite/PostgreSQL database selector.
Configure static and template directory paths.
[NEW] 
models.py
User: Custom User class inheriting from AbstractUser.
Goal: Name, Target Date, Priority (High/Medium/Low), Daily Study Hours, Is Completed, Created At.
Task: Title, Goal (nullable), Due Date, Status (Pending, In Progress, Completed), Priority, Created At.
PlannerSettings: Wake Up Time, Sleep Time, College/Office Start Time, College/Office End Time, Free Hours (extra), Weekend Availability.
ScheduleItem: Title, Start Time, End Time, Type (Sleep, Exercise, Work/College, Rest, Study, Revision, Other), Day of Week (or Is Weekend).
Habit: Name, Category (Reading, Exercise, Coding, Water, Sleep, Meditation), Created At.
HabitLog: Habit (FK), Date, Completed (Boolean), Value (Optional).
[NEW] 
serializers.py
Define serializers for User (Registration), Goal, Task, PlannerSettings, ScheduleItem, Habit, and HabitLog.
[NEW] 
utils.py
Smart Planner Algorithm: Algorithm to automatically partition a 24-hour day into a balanced schedule based on user's PlannerSettings and active goals.
Generates Sleep block first.
Fills Work/College timings next.
Inserts Morning/Evening exercise and relaxation routines.
Distributes Study hours and a dedicated Revision block from remaining free slots.
[NEW] 
views.py
DRF API views for:
Auth: Register, Login (Token Obtain), Refresh, User Profile.
Goals: CRUD endpoints.
Tasks: CRUD endpoints (with filters).
Planner: Settings view + Auto-generate Schedule trigger.
Habits: List, Create, Delete + Log completion.
Analytics: Aggregated reports for completed tasks, habit streaks, daily and weekly study hours.
Django HTML Template views for:
